# Academic research strategy

Use this when a request needs more than a single known-paper lookup.

## Start with the research question

Translate the request into concepts, population/domain, method, outcome, and time boundary as
applicable. Search the distinctive concepts first. Add filters only when they express the
question; over-filtering early hides vocabulary and indexing differences.

Build query variants from:

- canonical terminology and abbreviations
- older or neighboring terminology
- task/problem terms, not just a favored method
- benchmark, survey, review, limitation, and negative-result language
- adjacent fields that may describe the same mechanism differently

Do not set a generic number of queries or papers. Stop when added variants and graph expansion
produce diminishing relevant results relative to the requested confidence and scope.

## Use sources deliberately

Start with unified search. Then select provider-specific commands for a reason:

- S2 recommendations for conceptually related work around a strong seed
- S2 citations/references for forward and backward chaining
- S2 snippets to locate candidate passages before opening full text
- OA structured filters for institutions, venues/sources, topics, types, and date ordering
- OA group-by for corpus-level distributions, not claims about individual papers
- ChinaRxiv for current Chinese preprints that may not yet be represented elsewhere

Provider counts are not interchangeable. Record the source and retrieval boundary when
reporting totals or rankings.

## Expand through the citation graph

Choose seeds because they are central, recent, methodologically relevant, or strong surveys;
do not choose only by citation count. For each useful seed, inspect:

1. references for foundations and prior methods
2. citations for follow-up work, replications, and critiques
3. recommendations for terminology-mismatched neighbors

Deduplicate by DOI first, then arXiv/provider IDs, then normalized title with manual review.
Preprint and published versions may represent the same work; prefer the version appropriate to
the user's purpose and note meaningful version differences.

## Search the local corpus

Download papers only when access and user intent permit. Use hybrid search for most questions,
FTS for exact terminology, and semantic mode for paraphrases. Passage hits are navigation aids:
read surrounding text, methods, tables, and limitations before extracting a claim.

For cross-paper synthesis, track for each paper:

- question and setting
- data/sample and evaluation design
- intervention/model and comparator
- result with uncertainty or conditions
- limitations and conflicts
- stable identifier and version

## Handle negative and absence claims

"No papers found" is not "no papers exist." Before making an absence claim:

- search synonyms, acronyms, and broader task language
- search both unified and relevant provider-specific indexes
- traverse citations from nearby work
- check whether filters, date boundaries, or failed providers removed candidates
- state the databases, terms, and limitations of the search

Use calibrated language such as "I did not find ... in the searched sources" unless the
evidence justifies a stronger conclusion.

## Produce an auditable answer

Separate bibliographic facts from your synthesis. Cite primary papers for technical claims and
surveys for landscape framing. Include identifiers or links the user can resolve. When metadata
or citation counts disagree, prefer the publisher/DOI record for publication facts and label
provider-specific metrics rather than silently merging them.
